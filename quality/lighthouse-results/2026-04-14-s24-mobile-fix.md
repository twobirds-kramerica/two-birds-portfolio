# S24 Mobile Layout Fix -- 2026-04-14

## Audit Findings

The CSS architecture was fundamentally sound. `overflow-x: hidden` was set on html, body, and page-wrapper. All images had `max-width: 100%`. No inline styles broke the viewport.

**Root cause identified:** The `.sidebar` element had `position: fixed; left: 0; width: 280px` with no CSS rule to hide it off-screen on mobile. On desktop this works because the sidebar is always visible. On mobile (360-430px), the sidebar was rendering at `left: 0` and overlaying the top-left content area, creating the "left skew" appearance.

**Secondary factors:**
- No dedicated mobile breakpoint between 360px and 480px existed for structural layout
- `.main-content` used `padding: 36px 24px` which is fine for content width but the container wasn't explicitly capped at `100vw`

## Fix Applied (commit `3f8a777`)

Added `@media (max-width: 430px)` block to `css/main.css`:

1. **Sidebar:** `transform: translateX(-100%)` by default on mobile; `transform: translateX(0)` when `.open` class is present. This hides the sidebar off-screen until the hamburger is tapped.
2. **All containers:** explicit `width: 100%; max-width: 100vw; box-sizing: border-box`
3. **Main content:** `padding-left: 16px; padding-right: 16px` (down from 24px for more content space)
4. **Top bar and location bar:** capped at `100vw` with `box-sizing: border-box`
5. **Tables:** `display: block; overflow-x: auto` to prevent viewport breakout
6. **Pre/code blocks:** `overflow-x: auto; word-wrap: break-word`
