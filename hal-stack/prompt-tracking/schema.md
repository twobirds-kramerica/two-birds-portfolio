# Prompt Tracking Schema

## Purpose
Track which Claude Code build prompts have been given, run, completed, and failed.
Solves the core problem: Aaron loses track of what Claude Web generated vs what Claude Code actually ran.

## Current Solution (Manual — Gmail labels)
1. Claude Web generates prompt → saves Gmail draft with subject prefix "⏳ UNCONFIRMED PROMPT"
2. Aaron pastes prompt into Claude Code → screenshots result
3. Aaron sends screenshot to Claude Web → draft updated to "✅ CONFIRMED"

## Target Solution (Automated — n8n + SQLite)
Table: prompts
- id: auto-increment
- prompt_id: UUID
- generated_at: timestamp
- topic_summary: string (max 100 chars)
- full_prompt: text
- status: PENDING | RUNNING | COMPLETE | FAILED | PARTIAL
- started_at: timestamp (null until running)
- completed_at: timestamp (null until done)
- github_commit: string (null until pushed)
- error_summary: text (null unless failed)
- phases_total: int
- phases_complete: int

## n8n Workflow — Prompt Lifecycle
**Trigger:** Gmail label PROMPT-QUEUE applied to draft
- Action 1: Write row to SQLite (status: PENDING)
- Action 2: Send confirmation to aaron.patzalek@gmail.com

**Trigger:** GitHub push event (via webhook)
- Action 1: Match commit message to pending prompt
- Action 2: Update row (status: COMPLETE, github_commit: [hash])

**Trigger:** Claude Code error output detected
- Action 1: Update row (status: FAILED, error_summary: [error])
- Action 2: Email alert to aaron.patzalek@gmail.com

## Gmail Label Convention (Interim System)
| Label | Meaning |
|-------|---------|
| ⏳ UNCONFIRMED PROMPT | Generated by Claude Web, not yet run in Claude Code |
| 🔄 IN PROGRESS | Aaron has pasted it into Claude Code |
| ✅ CONFIRMED COMPLETE | Build ran, pushed to GitHub |
| ❌ FAILED | Error encountered, needs retry or redesign |
| 📦 ARCHIVED | Completed and no longer active |
