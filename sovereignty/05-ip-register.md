## 3. Frameworks and Methodologies

### Trust-First Design
- Core principle: show users how to leave before showing them how to stay
- Applied throughout DCC — Module 1 is "Mastering the Escape Hatch"
- Transferable to any product targeting anxious or novice users

### Sovereignty Doctrine
- "Use the best tools. Own the exit door."
- Framework for vendor dependency management
- Documented in this sovereignty/ directory

### Brenda Persona
- Fictional QA character: Brenda, 74, St. Thomas, Ontario
- Uses iPad only, reads at 150% zoom, confused by technical jargon
- Every DCC feature must pass the "Would Brenda understand this?" test

### HAL Stack
- Infrastructure-as-code philosophy for solo operators
- Components: prompt tracking, session state, overnight builds, quality monitoring
- Documented in `hal-stack/` directory
